# 신규 도로/터널 후보 분석 파이프라인

## MVP 흐름

1. 사용자가 출발지와 도착지 좌표를 입력합니다.
2. 각 좌표에서 가장 가까운 기존 도로 노드를 찾습니다.
3. 두 접속 노드 사이에 신규 후보 노선을 생성합니다.
   - `straight`
   - `left_bypass`
   - `right_bypass`
4. 후보 노선을 일정 간격으로 샘플링합니다.
5. 각 샘플 지점의 DEM 고도를 조회합니다.
6. 인접 샘플 사이 경사도를 계산합니다.
7. 경사도 기준으로 일반 도로, 급경사 도로, 터널 후보를 분리합니다.
8. 후보별 공사비를 계산하고 가장 낮은 비용 후보를 추천합니다.

## API 분리

### `POST /api/evaluate-route`

후보 노선 생성 알고리즘이 만든 좌표 배열을 받아 DEM 기반으로 평가합니다. 후보 생성은 하지 않습니다.

```json
{
  "name": "candidate_a",
  "coordinates": [
    { "lat": 37.5665, "lon": 126.978 },
    { "lat": 37.61, "lon": 127.02 },
    { "lat": 37.6542, "lon": 127.0568 }
  ],
  "sample_interval_m": 90,
  "rock_factor": 1.15
}
```

### `POST /api/route-cost`

현재 MVP용 임시 API입니다. 출발지와 도착지에서 가까운 기존 도로 접속점을 찾고, 백엔드가 단순 후보 노선 3개를 만든 뒤 내부적으로 `evaluate-route`와 같은 평가 로직을 사용합니다.

### `POST /api/generate-route`

후보 노선 알고리즘 전용 API입니다. 출발지와 도착지 좌표를 받아 후보 노선 좌표 배열만 반환합니다. 평가와 비용 산정은 하지 않습니다.

## 현재 적용한 판정 기준

- 경사도 0~8%: 일반 도로
- 경사도 8~15%: 급경사 도로
- 경사도 15% 이상: 터널 후보
- 터널 후보가 300m 미만이면 급경사 도로로 낮춰 봅니다.
- 터널 후보 사이의 비터널 구간이 100m 이하이면 하나의 터널로 병합합니다.

## 한계

- 현재 후보 노선은 단순 직선/좌우 우회선입니다.
- 환경영향, 토지이용, 하천, 기존 구조물, 법정 도로 설계 기준은 아직 반영하지 않습니다.
- DEM 90m 해상도 기반이므로 정밀 설계가 아니라 후보 비교용입니다.
